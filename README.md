# URL SHORTENER

## Общее описание
Превратите любую длинную и неудобную ссылку в короткую и элегантную за секунду - просто вставьте URL и получите компактную ссылку, которая идеально впишется в любое сообщение, пост или рекламу.
Ваши ссылки автоматически очищаются через 5 минут бездействия, что означает полную конфиденциальность и отсутствие цифрового мусора в интернете - никто не сможет найти ваши старые ссылки спустя
время.

Это сервис на базе Key-Value Storage (аналог Redis). Сервис для создания коротких и удобных ссылок из длинных веб-адресов. Вы даете длинную ссылку - получаете короткую, которая ведет в то же место.

Принимает команды (SET, GET, DEL, ...). Данные хранятся в памяти в HashMap. Реализовано TTL.

Как это работает?

1. Отправьте длинную ссылку - например, https://very-long-website-url.com/article/123/some-title
2. Получите короткую - например, http://localhost:3000/2Jrvng
3. Используйте короткую ссылку - она автоматически перенаправит на оригинальный адрес


Разработка ведется по этапам. От простого к сложному. После выполнения каждого этапа создаетсчя ПР на проверку + добавляется более подробное описание работы, сделанной на этапе.

## Этап 1. Парсер команд, хранилище
На этом этапе необходимо реализовать главную логику с принятем команд и хранением данных. Взаиможействие с кодом осуществляется путем unit-тестов и вызовами в main.

Должны быть реализованы следующие команды:
* Set         - Сохранение пары ключ-значение
* Get         - Получение значения по rj.xe
* Delete      - Удаление по ключу
* Exists      - Проверка на существование значения
* SetWithTTL  - Сохранение пары ключ-значение c временем жизни значения (TTL)
* Info        - Информация о хранилище
* Keys        - Получить список ключей (анадлог KEYS *)
* Clear       - Очистка хранилища

## Этап 2. Логика конвертера ссылок
Должны быть реализована следующая команда:
* SetUrl - Сохранение ссылки с получением короткого ключа до нее

Реализована логика в converter.rs. При сипользовании метода SetUrl передаваемая строка сохраняется в value, а в качестве ключа используется захешированная строка (хэширование и base62).

Команда работает так: SETURL https://example.com вернет короткую строку, которая будет сохранена как ключ со значением исходного URL.

## Этап 3. TCP сервер с использованием Tokio для взаиможействия с сервисом
Добавлена новая команда SetTTL.

Использование:
* Set mykey "hello" - создаем ключ
* SetTTL mykey 60   - устанавливаем TTL 60 секунд
* SetTTL mykey 0    - удаляем ключ немедленно

Так же был коздан controller с двумя эндпоинтами:
- POST /getShortUrl - принимает JSON {url: "..."}, возвращает {short_url: "http://localhost:3000/[shortUrl]"}
- GET /[shortUrl] - редиректит на оригинальный URL

Логика работы TTL:
1. При создании короткого URL: Устанавливается TTL 5 минут
2. При каждом использовании: TTL продлевается на 5 минут
3. Неиспользуемые URL: Автоматически удаляются через 5 минут после последнего обращения
4. Активно используемые URL: Остаются доступными пока их используются
